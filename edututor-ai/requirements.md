# EduTutor AI — 功能需求文档

> 版本：v1.0.0 | 最后更新：2026-07-07

---

## 1. 项目概述

EduTutor AI 是一个基于 Spring AI 2.0 的智能教育 RAG 助手，面向教育培训机构、学校和企业内训场景，提供 7×24 智能助教、课后答疑、测验生成和学习路径规划。

### 1.1 核心目标

| 目标 | 描述 | 度量 |
|------|------|------|
| RAG 准确率 | 基于课程资料的问答准确率 | ≥ 90% Top-3 召回 |
| 响应延迟 | 学生提问到回答的端到端延迟 | P95 < 5s |
| 多租户隔离 | 不同课程/机构的知识库完全隔离 | 零交叉泄漏 |
| 测验质量 | AI 生成测验与人工出题的一致性 | ≥ 85% 教师认可率 |

### 1.2 用户角色

| 角色 | 权限 | 场景 |
|------|------|------|
| 学生 (Student) | 提问、参加测验、查看学习路径 | 课后自学 |
| 教师 (Teacher) | 审核问答、管理题库、生成测验 | 教学辅助 |
| 管理员 (Admin) | 知识库管理、租户配置、监控 | 系统运维 |

## 2. 功能需求

### 2.1 智能问答 (Agent Ask)

**FR-01** 学生可通过 REST API 提交自然语言问题。
**FR-02** 系统自动检索课程资料知识库，返回带引用来源的回答。
**FR-03** 支持多轮对话，保持会话上下文（sessionId）。
**FR-04** 若知识库无法覆盖问题，应明确告知而非编造答案。

### 2.2 测验生成 (Quiz Generator)

**FR-05** 支持按知识点/章节/难度生成单选、多选、判断题。
**FR-06** 每道题必须附带正确答案和解析。
**FR-07** 支持按 Bloom 分类法设置认知层级（记忆/理解/应用/分析/评价/创造）。
**FR-08** 教师可审核、编辑、批准 AI 生成的题目后入库。

### 2.3 学习路径规划 (Learning Path)

**FR-09** 根据学生当前水平和目标，生成个性化学习路径。
**FR-10** 路径包含：前置知识 → 核心章节 → 练习 → 里程碑测验。
**FR-11** 支持生成学习时间估算和每日建议学习量。

### 2.4 作业提示 (Homework Hint)

**FR-12** 学生提交作业问题时，提供引导性提示而非直接答案。
**FR-13** 提示分三级：概念回顾 → 思路引导 → 分步解析（教师可配置）。

### 2.5 学习进度报告 (Progress Report)

**FR-14** 自动追踪学生在各知识点的掌握程度。
**FR-15** 生成可视化的学习报告（掌握度雷达图、薄弱点列表）。
**FR-16** 支持导出 PDF 格式报告。

### 2.6 教师审核队列 (Teacher Review)

**FR-17** AI 回答置信度低于阈值时，自动加入教师审核队列。
**FR-18** 教师可批注/修正后发布，修正结果反哺 RAG 知识库。

### 2.7 知识库管理 (KB Management)

**FR-19** 支持上传 PPT/PDF/DOCX/MD 格式课程资料。
**FR-20** 自动分块 → 向量化 → 入库（PGVector）。
**FR-21** 支持增量同步和全量重建索引。
**FR-22** 支持按课程/章节/标签组织知识库。

## 3. 非功能需求

### 3.1 性能

| 指标 | 目标 |
|------|------|
| 问答 API P50 延迟 | < 2s |
| 问答 API P95 延迟 | < 5s |
| 知识库同步吞吐量 | ≥ 10 MB/min |
| 并发学生数 | ≥ 100 (单节点) |

### 3.2 安全

- **NFR-01** 多租户数据隔离：Tenant ID 注入所有 SQL/RAG 查询
- **NFR-02** 输入过滤：防 Prompt Injection，敏感词过滤
- **NFR-03** API Key / 数据库密码不入库，走 `.env` + 环境变量
- **NFR-04** 学生个人数据匿名化存储

### 3.3 可观测性

- **NFR-05** Prometheus metrics：请求量、延迟、错误率、Token 消耗
- **NFR-06** OpenTelemetry 链路追踪（Spring Actuator 集成）
- **NFR-07** 结构化日志（JSON 格式）

### 3.4 可扩展性

- **NFR-08** 工具插件化：新增 Tool 只需添加 `@Tool` 注解
- **NFR-09** 模型切换：支持 Ollama / OpenAI / 阿里百炼 热切换
- **NFR-10** 向量数据库可替换：默认 PGVector，可扩展至 Milvus / Elasticsearch

### 4.2 数据流

```text
1. 学生提问 → POST /api/agent/ask
2. TenantContextFilter 注入 tenantId
3. ChatClient 接收 question + sessionId
4. Agent 决策 → 调用 @Tool course_material_search
5. RAG Service → PGVector 向量检索 (tenantId 过滤)
6. 检索结果 + 原始问题 → LLM 生成回答 (引用来源)
7. 置信度检查 → 低于阈值 → 入教师审核队列
8. 响应返回学生 (含引用链接)
```

### 4.3 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 框架 | Spring Boot 3.x + Spring AI 2.0 | Agent 编排与 Tool Calling |
| 数据库 | PostgreSQL 16 + PGVector | 向量存储 + 业务数据 |
| LLM | Ollama (qwen2.5:7b) | 本地默认模型 |
| Embedding | mxbai-embed-large | 文本向量化 |
| 迁移 | Flyway | 数据库版本管理 |
| 容器 | Docker Compose | 一键启动 PG + 应用 |
| 监控 | Prometheus + Actuator | 指标采集 |
| 链路 | OpenTelemetry | 分布式追踪 |
| 构建 | Maven | Java 项目构建 |

## 5. API 接口

### 5.1 核心 API

| Method | Path | Auth | 描述 |
|--------|------|------|------|
| POST | `/api/agent/ask` | Tenant Header | 学生问答 |
| POST | `/api/quizzes/generate` | Tenant + Role | 生成测验 |
| POST | `/api/learning-path/plan` | Tenant + Role | 学习路径 |
| POST | `/api/kb/sync` | Admin | 同步知识库 |
| GET | `/api/kb/search?q=` | Tenant Header | 检索知识库 |
| POST | `/api/homework/hint` | Tenant Header | 作业提示 |
| GET | `/api/progress/{userId}` | Tenant + Auth | 学习报告 |
| GET | `/api/review/queue` | Teacher | 审核队列 |

### 5.2 请求/响应格式

```json
// POST /api/agent/ask
{
  "question": "向量数据库和传统数据库的区别是什么？",
  "userId": "stu_1001",
  "sessionId": "sess_20260707"
}

// Response
{
  "answer": "向量数据库的核心区别在于...",
  "sources": [
    {"title": "数据库原理-第5章", "chunk": "ch5_p3", "relevance": 0.94}
  ],
  "confidence": 0.92,
  "sessionId": "sess_20260707"
}
```
